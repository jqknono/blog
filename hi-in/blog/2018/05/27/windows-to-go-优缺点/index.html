Windows To Go के फायदे और नुकसान

Sunday, May 27, 2018
Categories:
  • मार्कडाउन

Windows To Go

Windows To Go का फायदा इसकी पोर्टेबिलिटी (portability) है, और नुकसान यह है कि क्लासिक Windows सिस्टम की कुछ सुविधाएं सीमित हैं।

प्रस्तावना

Windows To Go काफी समय से मौजूद है, लेकिन Baidu पर मिलने वाले चीनी दस्तावेज़ इतने कम हैं, जिससे घरेलू आईटी तकनीक के विकास के लिए चिंता बढ़ जाती है। लेखक J ने काम करने का समय ज्यादा नहीं हुआ है और उनकी क्षमता सीमित है, लेकिन काम में उन्हें बड़ी मात्रा में अंग्रेजी विकास दस्तावेज़ों से निपटना पड़ता है। इसलिए, वे अभी भी आशा करते हैं कि बुनियादी मार्ग प्रशस्त करने का काम कर सकें ताकि आने वाले लोगों के लिए संदर्भ लेना आसान हो सके। कृपया पाठक यदि कोई त्रुटि हो तो बिना संकोच की बताएं। Windows To Go के लिए विस्तृत आधिकारिक दस्तावेज़ उपलब्ध हैं, जिन्हें अंग्रेजी पढ़ने में सक्षम लोग सीधे माइक्रोसॉफ्ट के आधिकारिक दस्तावेज़ों पर जा सकते हैं। लिंक नीचे दिए गए हैं:

  • Windows To Go Overview
  • Best practice recommendations for Windows To Go
  • Deployment considerations for Windows To Go
  • Prepare your organization for Windows To Go
  • Security and data protection considerations for Windows To Go
  • Windows To Go: frequently asked questions

यह लेख मुख्य रूप से अवलोकन (Overview) और कुछ सामान्य प्रश्नों का परिचय देगा, अधिकांश सामग्री अनुवादित है, और लेखक की कुछ चेतावनियों को [J] से चिह्नित किया गया है ताकि यह सुनिश्चित हो सके कि पाठक को गुमराह न किया जाए।

Windows To Go का अवलोकन

Windows To Go Windows Enterprise और Education संस्करणों पर एक सुविधा है, ज्यादातर घरेलू उपयोगकर्ताओं द्वारा उपयोग किए जाने वाले Home संस्करण में यह सुविधा नहीं है। यह हमें USB ड्राइव या हार्ड डिस्क से बूट होने वाला पोर्टेबल Windows सिस्टम बनाने की अनुमति देता है। Windows To Go पारंपरिक कार्य उपकरणों को बदलने के लिए नहीं बनाया गया था। इसका मुख्य उद्देश्य उन लोगों को अधिक कुशल बनाना है जिन्हें अक्सर वर्कस्पेस बदलने की आवश्यकता होती है। Windows To Go का उपयोग शुरू करने से पहले, उपयोगकर्ताओं को निम्नलिखित सावधानियों को समझना होगा:

  • Windows To Go और पारंपरिक Windows स्थापना विधियों के बीच अंतर;
  • Windows To Go का उपयोग करके मोबाइल वर्क करना;
  • Windows To Go स्थापित करने की तैयारी;
  • हार्डवेयर आवश्यकताएं।

Windows To Go और पारंपरिक Windows स्थापना विधियों के बीच अंतर

Windows To Go का कार्य वातावरण पारंपरिक Windows के समान ही है, केवल निम्नलिखित अंतर हैं:

  • उपयोग में आने वाले USB ड्राइव को छोड़कर, मशीन के अन्य हार्ड डिस्क डिफ़ॉल्ट रूप से ऑफ़लाइन अवस्था में होते हैं। यानी फ़ाइल एक्सप्लोरर में वे दिखाई नहीं देते, यह डेटा की सुरक्षा के लिए है। [J] लेकिन फिर भी आपके पास अन्य हार्ड डिस्क को प्रदर्शित करने और उसमें फ़ाइलों को संशोधित करने के तरीके हैं।
  • TPM (Trusted Platform Module) उपलब्ध नहीं है। TPM मॉड्यूल किसी विशेष कंप्यूटर से बांधा जाता है, जो व्यावसायिक डेटा की सुरक्षा करता है। [J] अधिकांश नागरिक कंप्यूटरों में TPM मॉड्यूल नहीं होता है, लेकिन यदि आपका व्यावसायिक कंप्यूटर पहले से ही अपनी कंपनी के डोमेन में शामिल हो चुका है, तो उस कंप्यूटर पर Windows To Go का उपयोग करने का प्रयास न करें, अन्यथा सलाह दी जाती है कि आप पहले से ही अपनी अगली नौकरी के लिए रिज्यूमे तैयार कर लें।
  • Windows To Go में हाइबरनेशन डिफ़ॉल्ट रूप से अक्षम है, लेकिन इसे ग्रुप पॉलिसी के माध्यम से चालू किया जा सकता है। [J] कई मशीनें हाइबरनेशन के दौरान USB डिवाइस के साथ कनेक्शन बंद कर देती हैं, जिसके परिणामस्वरूप हाइबरनेशन से पुनर्प्राप्ति नहीं हो पाती, यह समझना बहुत आसान है, माइक्रोसॉफ्ट ने हमारे लिए इस बात का ध्यान रखा है, इसलिए इस सेटिंग को बदलने की कोई आवश्यकता नहीं है।
  • Windows की पुनर्प्राप्ति (Restore) सुविधा अक्षम है। यदि सिस्टम में कोई समस्या आती है, तो केवल Windows को पुनर्स्थापित (reinstall) किया जा सकता है।
  • फ़ैक्टरी सेटिंग्स में पुनर्प्राप्ति उपलब्ध नहीं है, Windows को रीसेट करना उपलब्ध नहीं है।
  • अपग्रेड उपलब्ध नहीं है। Windows को स्थापना के समय के संस्करण पर ही रहना होगा, Windows 7 से 8 में अपग्रेड नहीं किया जा सकता, और न ही Windows 10 Red Stone 1 से Red Stone 2 में अपग्रेड किया जा सकता है।

Windows To Go का उपयोग करके मोबाइल वर्क करना

Windows To Go कई मशीनों के बीच स्विच किया जा सकता है, सिस्टम स्वचालित रूप से तय करता है कि डिवाइस बूट करने के लिए कौन से ड्राइवरों की आवश्यकता है। कुछ एप्लिकेशन जो सिस्टम हार्डवेयर के साथ दृढ़ता से जुड़े हुए हैं, शायद काम न करें। [J] जैसे ThinkPad टचपैड सेटिंग प्रोग्राम, फिंगरप्रिंट पहचान सेटिंग प्रोग्राम आदि।

Windows To Go स्थापित करने की तैयारी

आप System Center Configuration Manager या Windows के मानक डिप्लॉयमेंट टूल जैसे DiskPart, Deployment Image Servicing and Management (DISM) का उपयोग कर सकते हैं। निम्नलिखित मुद्दों पर ध्यान दें:

  • क्या Windows To Go इमेज में इंजेक्ट करने के लिए कोई ड्राइवर है?
  • अलग-अलग मशीनों पर काम करते समय, डेटा को कैसे उचित रूप से संग्रहीत और सिंक्रनाइज़ किया जाए?
  • 32-बिट या 64-बिट? [J] नई मशीनें 64-बिट का समर्थन करती हैं, 64-बिट प्रोसेसर वाली मशीनें 32-बिट सिस्टम भी चला सकती हैं, 32-बिट प्रोसेसर 64-बिट सिस्टम नहीं चला सकते, 64-बिट सिस्टम चलने पर अधिक हार्ड डिस्क स्पेस और मेमोरी स्पेस का उपयोग होता है। यदि आपको उपयोग की जाने वाली मशीन के प्रोसेसर आर्किटेक्चर में केवल 32-बिट प्रोसेसर का समर्थन करने वाले या मशीन की मेमोरी 4GB से कम है, तो आपको 32-बिट सिस्टम का उपयोग करने की सलाह दी जाती है।
  • सहयोगी नेटवर्क के बाहर के नेटवर्क से रिमोट कनेक्ट होने पर रिज़ॉल्यूशन कितना सेट किया जाना चाहिए?

हार्डवेयर आवश्यकताएं

USB हार्ड डिस्क या USB ड्राइव

Windows To Go ने नीचे सूचीबद्ध उपकरणों के लिए आवश्यकताओं को पूरा करने के लिए विशेष रूप से अनुकूलन किया है, जिसमें शामिल है:

  • USB उपकरणों पर उच्च यादृच्छिक रीड/राइट को अनुकूलित करना, ताकि दैनिक संचालन अधिक सुचारू हो।
  • प्रमाणित उपकरणों पर Windows 7 और बाद के सिस्टम बूट हो सकते हैं।
  • भले ही Windows To Go चल रहा हो, USB उपकरणों को अभी भी निर्माता की वारंटी समर्थन प्राप्त है। [J] इसका मतलब यह नहीं कि जिस कंप्यूटर में USB ड्राइव लगाया गया है वह वारंटी के दायरे में आएगा।

प्रमाणित न होने वाले USB उपकरणों पर Windows To Go का उपयोग समर्थित नहीं है। [J] इस्तेमाल हो सकता है या नहीं, यह आज़माकर पता लगाया जा सकता है, और अगर नहीं चला तो पता भी चल जाएगा कि क्यों। [J] साथ ही, इंटरनेट पर USB ड्राइव के निर्माता और मॉडल को संशोधित करके जबरन समर्थन प्राप्त करने के वैकल्पिक तरीके भी हैं, जिनका विवरण यहाँ नहीं दिया गया है।

होस्ट कंप्यूटर (Host computer)

  • Windows 7 और बाद के सिस्टम का समर्थन करने के लिए प्रमाणित।
  • Windows RT चलाने वाले कंप्यूटर समर्थित नहीं हैं।
  • Apple Mac कंप्यूटर समर्थित नहीं हैं। [J] हालांकि इंटरनेट पर Windows To Go को Mac पर चलाने के अनुभव के बारे में बातचीत फैली हुई है, लेकिन आधिकारिक दस्तावेज़ों में स्पष्ट रूप से कहा गया है कि Mac के उपयोग के परिदृश्य समर्थित नहीं हैं।
